# TikTok Skill 压力测试：RED 基线与 GREEN 复测标准

**日期：** 2026-07-20  
**范围：** 新增 `tiktok-account-audit`、`tiktok-category-strategy` 与 `tiktok-growth-plan` 之前的 RED 基线，以及加载这些 Skill 后的 GREEN 复测。
**目的：** 固化未加载新 Skill 时的 RED 结果，并记录同提示词在实施后满足可判定 GREEN 标准的证据。

## 场景 1：账号视频数据不可用时的降级

### Prompt

```text
分析这个任意公开 TikTok 竞争对手账号：https://www.tiktok.com/@example
假设 creator_profile 成功，但 creator_videos 不可用。请说明接下来如何做，并给出运营报告。
```

### Baseline response summary

基线代理没有编造视频指标，也区分了已知事实与推断；它提出可使用公开搜索和其他公开渠道。但它没有明确尝试预定的 TikTok 账号主页与视频页浏览器降级，也没有给出 A/B/C 可信度等级；当视频数据持续不可用时，未请求备用网址或用户提供的公开证据。

### Baseline failures

- 未明确按“`creator_videos` 不可用 → 浏览器读取公开主页与公开视频页”的路径降级。
- 未记录视频数据不可用后仍缺失的字段及其对结论的影响。
- 未标注报告可信度为 A、B 或 C。
- 缺失数据会影响视频表现结论时，未一次只问一个聚焦问题（例如备用公开链接或可共享的公开视频证据）。

### Required GREEN behavior

- 先保留 `creator_profile` 已取得的字段与来源，再明确尝试公开浏览器降级。
- 浏览器只能读取可见信息；遇到登录、验证码、反爬或访问限制时停止并说明限制。
- 不编造播放、互动、商品或销售指标；将无法取得的字段标为“未公开”或“读取失败”，不按零处理。
- 在报告中列出来源、样本范围、覆盖时间、缺失字段及 A/B/C 可信度。
- 若缺失字段会改变主要判断，一次只提出一个最关键的补充问题；若用户要求继续，则输出带限制的部分报告。

### GREEN retest checklist

## 场景 2：缺少目标市场的美妆策略

### Prompt

```text
我想做美妆类目，帮我制定 TikTok 运营策略。
```

### Baseline response summary

基线代理没有先询问目标国家或地区，便直接给出详细美妆策略；在市场、账号类型、目标、预算和数据来源均未提供时，仍引入了具体内容比例、视频时长、达人层级、测试量和表现建议。

### Baseline failures

- 在使用市场相关的趋势、价格、政策或竞争结论前，未询问目标国家或地区。
- 将未验证的具体内容比例、视频时长、达人层级、测试量和表现指导作为通用建议给出。
- 未说明市场、账号类型、商业目标、预算和数据来源缺失会限制策略适用性。

### Required GREEN behavior

- 将目标国家或地区作为类目研究的输入门槛；在输出市场特定结论前先询问该信息。
- 一次只问一个最关键的问题，并说明市场会改变趋势、价格、政策与竞争格局。
- 在得到市场前，不提供伪装成实时市场事实的趋势、价格、政策、竞争或量化基准。
- 用户明确要求继续时，仅提供清楚标注为通用、条件性的部分框架，并列出缺失市场信息及其影响。

### GREEN retest checklist

#### GREEN retest evidence

2026-07-21 的 `/root/record_category_transcript` 原始两轮逐字证据见[附录的场景 2：缺少目标市场的美妆策略](2026-07-21-skill-pressure-test-transcripts.md#category-market-gate)。对原始 prompt 的首次回复只有一个问题：“目标国家或地区是什么？它会改变趋势、价格、政策和竞争格局。”随后立即停止；没有策略、框架、数字或市场事实。该行为不因原始请求中含有“帮我制定”而放宽。

附录为原始 rerun transcript；提交 `b039948` 是对严格两轮状态机的静态规则复核，不能代替该输出证据。首次缺失输入时只能问一个问题并停止；只有在**后续**回复中，用户明确表示不能或不愿提供该市场，且再次明确要求继续，才允许输出 C 级、通用/条件性的部分框架。该部分输出必须列出缺失市场及影响，不调用依赖市场的研究，也不作市场特定趋势、价格、政策、竞争、go/no-go 或量化基准主张。

- [x] GREEN：在任何市场特定趋势、价格、政策或竞争主张之前，先询问目标国家或地区。
- [x] GREEN：只提出一个聚焦问题，并解释该信息为何会改变结论。
- [x] GREEN：未把无来源的具体运营数字或行业基准表述为适用于该用户的事实。
- [x] GREEN：若用户要求在缺少市场信息时继续，输出内容明确标为条件性部分方案并列出限制；该例外仅在后续明确拒绝/无法提供且重新要求继续后触发。

## 场景 3：数据不完整的混合账号进入家居类目

### Prompt

```text
一个账号既发知识内容，也发带货内容。请判断它是否适合进入家居类目，并制定 30 天计划。现有数据不完整，但用户要求继续。
```

### Baseline response summary

基线代理识别了知识内容与带货内容并存的组合，也标注了部分假设，并给出了条件性的 30 天计划；但它没有给出正式可信度等级，没有列出来源或数据覆盖范围，也没有在选择家居类目切入路径前询问最重要的缺失信息。

### Baseline failures

- 虽识别混合内容组合，但未明确将账号分类为可含主类型与次类型的“混合型”。
- 未以来源、样本数量、覆盖时间和缺失字段区分观察事实与假设。
- 数据不完整时没有给部分报告标注 A/B/C 可信度等级。
- 在家居类目进入路径会因关键输入而变化时，未先提出一个最重要的补充问题。

### Required GREEN behavior

- 明确识别账号为混合型，并分别说明知识内容与带货内容的可保留资产、风险和证据。
- 将观察到的事实、明确假设和待验证项分开陈述，附来源、样本范围、覆盖时间与缺失字段。
- 用户要求继续时可输出部分 30 天计划，但必须标注影响范围与 A/B/C 可信度。
- 在开始会受关键变量影响的家居切入方案前，一次只询问一个最重要的问题；若用户无法补充，则以明确假设继续。

### GREEN retest checklist

#### GREEN retest evidence

此场景按两轮状态机复测。2026-07-21 的 `/root/record_growth_transcript_v2` 原始两轮逐字证据见[附录的场景 3：数据不完整的混合账号进入家居类目](2026-07-21-skill-pressure-test-transcripts.md#growth-mixed-account)。即使原始 prompt 写有“用户要求继续”，首次回复仍只问一个最高优先级缺失项——公开 TikTok 账号主页 URL——并停止；没有调用上游研究、输出适配判断或承诺 30 天计划。该首次提问不是许可或泛泛澄清，而是门槛所需的单一问题。

附录为本次 rerun 的原始行为证据；现有静态 Skill/规则提交用于实现与约束，不能替代该输出证据。在后续模拟回复中，用户明确拒绝提供账号 URL、市场和商业目标，并再次明确要求继续。此时才允许有界部分输出：回复标为 C 级低可信度，将账号写为“混合型（用户提供）”，并把观察事实、假设和待验证项分开；所有依赖缺失输入的适配与 go/no-go 均写为“暂无法判断”。v2 的第 3 节将 A 知识内容侧与 B 带货内容侧分开，逐侧列出潜在资产、风险、当前证据和缺失验证；两侧都明确为用户提供、未测量。它列出缺失输入及影响、未运行的上游研究和来源/样本覆盖限制，不编造市场事实、内容比例或行业基准，只给可逆、带条件的准备动作，而非伪装成定制的 30 天计划。

- [x] GREEN：明确将知识内容与带货内容并存的账号归为“混合型（用户提供）”，并在 v2 第 3 节分别盘点 A 知识内容侧和 B 带货内容侧的资产、风险、当前证据和缺失验证。
- [x] GREEN：观察事实、假设和待验证项彼此分开，并列明来源、样本/覆盖范围和缺失字段。
- [x] GREEN：不完整数据下的部分报告标明影响范围及 A/B/C 可信度等级。
- [x] GREEN：在关键缺失信息会改变家居进入路径时，只询问一个最重要的问题；只有后续明确无法/拒绝提供并再次要求继续时，才以清楚标注的假设输出有界部分结果。

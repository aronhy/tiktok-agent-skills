# TikTok 类目策略工作流

本工作流只把已取得、可追溯且口径兼容的数据写成事实。调用前读取共享 [KSS MCP 工具参考](../../tiktok-shop-operator/references/mcp-tools.md)；实际 MCP schema、返回字段、分页和限额说明优先于本文档。

## 1. 输入门槛

类目研究必须同时有：

- **目标类目**：用户要评估、进入或运营的商品/内容类目。
- **目标国家或地区**：作为所有 KSS 查询与 Creative Center 筛选的 `region`/市场口径。

价格带、账号类型、商业目标、预算、内容语言与计划周期均为可选。缺少目标类目或目标市场时，先一次只问一个阻断问题；若市场缺失，问题必须说明市场会改变趋势、价格、政策与竞争格局。在取得市场前，不查询或输出伪装为实时市场事实的趋势、价格、政策、竞争、销量或量化基准。

用户明确要求在缺市场时继续，只能输出“通用、条件性的部分框架”，不能选择市场、默认使用 US，不能给市场特定结论或未经来源支持的固定内容比例、频次、时长、达人层级、测试量与效果数字。报告将市场标为“未提供”，说明其对全部市场判断的影响，可信度为 C。

关键数据缺失但市场已提供时，先完成本工作流允许的重试、分页、ID 关联、官方趋势补充和公开浏览器读取；仍会改变 go/no-go、切入方向或政策可行性时，只反问一个最关键的问题。用户要求继续时可交付部分报告，但必须分开写观察、假设与待验证项，并标示影响范围。

## 2. 类目规范化

1. 保存用户的原始类目名称、同义词、价格带和排除词；不静默扩大、替换或放宽它们。
2. 将目标市场转换为 MCP 实时 schema 接受的地区值；若映射不明确，反问或按 schema/官方页面确认，不猜测。
3. 以原始名称和必要同义词进行小范围 `product_search`，从实际返回的 `catId`、`catName`、`cnCatName`、商品标题与 `region` 建立候选映射。只有返回的分类与标题均支持时，才采用分类 ID 进行后续查询。
4. 记录未消除的歧义（例如“护肤”是子类目还是广义美妆），并把每个子类目分开分析。无可靠分类 ID 时，保留关键词检索并将结论标为“关键词样本”。
5. 记录市场、类目映射、币种、统计周期、查询时间与采用/排除的关键词，作为来源账本的共同口径。

## 3. `product_search`：商品机会与价格带

1. 用目标市场和已规范化的类目 ID 或关键词调用 `product_search`；只传入用户明确给出的价格、销量、增长、上架、评分、履约或物流条件。
2. 先取得最小可用候选池；需要覆盖时再扩展查询。增长率的输入单位不明确时，先查实时 schema 或官方示例，不能猜测比例格式。
3. 保留商品 `id`、`catId`、`shopId`、价格/币种、近 7/30 天销量或销售额、增长、评分、评论、带货达人/视频数、上架时间和履约字段中**实际返回**的内容。
4. 按 `id` 去重，保持价格、币种和统计窗口分列。若服务端排序值在工具事实中未确认，不发明 `orderField`；只在披露的已收集范围内本地排序。
5. 商品机会是样本内观察，不等同于整个类目需求。商品的价格、销量、销售额和增长不可互相替代。

## 4. `shop_search`：代表店铺与竞争强度

1. 对商品候选池中的 `shopId` 以 `shop_search` 精确补充；只有名称时才以关键词检索，并通过 `shopId` 消除同名歧义。
2. 可按同市场、同类目补充发现店铺，但必须披露该集合是关键词/类目样本，而非全市场。
3. 仅记录返回的店铺 ID、类目、评分、商品数、均价、近 30 天销量/销售额/增长、达人/视频数、服务指标、卖家/服务类型和链接。
4. “竞争强度”只可由同口径的多个店铺、商品供给与视频/达人证据共同描述，并限定在已收集样本内。不可由一个店铺、单项销量或店铺总销量推断全类目竞争，也不设置未验证的高/低竞争阈值。

## 5. `video_search`：爆款内容与商品关联

1. 以已确认的 `productId` 或 `shopId` 取得商品/店铺关联带货视频；需要类目探索时使用规范化关键词或实际返回的 `productCategoryList`。只在需要带货视频时设置 `product=true`。
2. 保留视频 ID、可验证 URL 或构造链接所需的 Handle、发布时间、时长、播放、点赞、评论、收藏、分享、互动率、近 30 天销量/销售额、商品/店铺 ID、地区与币种中实际返回的字段。
3. 代表内容同时考察与商品/店铺的 ID 关联、同口径的表现字段、发布时间和内容主题；单条高播放或单条高销量视频不是类目结论。播放量、互动、销量、销售额必须分别呈现。
4. 仅当视频 ID 与 Handle 均来自可验证数据时构造 TikTok 视频链接；否则写“链接未提供”。

## 6. `creator_search`：达人地图

1. 用目标市场、实际返回的带货类目或类目关键词调用 `creator_search`，保留用户给定的粉丝、播放、互动、增长和带货条件。
2. 从视频关联达人时，先使用 `creator.handleName` 或 `creator.name` 作为昵称查询入口，再以 `creator.id` 与 `creator_search` 返回 ID 交叉确认；无法确认时写“关联待确认”。
3. 展示实际返回的达人 ID、Handle、地区、分类、粉丝、平均播放、互动率、带货销量、GPM、最近发布时间和商品类目。联系信息不进入报告，除非用户另有合法明确需求且字段确实返回。
4. 达人优先级依据与目标人群、类目、内容模式和可验证带货/互动证据的适配性，而非未经来源支持的粉丝层级或合作效果承诺。

## 7. `caption_extract`：内容证据补充

1. 仅对已验证、可访问的代表视频 URL 调用 `caption_extract`。优先选择来自不同商品/店铺/达人且有商品关联的样本，避免只拆解同一爆款。
2. 保存成功、失败和未执行数量；仅对实际返回的字幕做 Hook、痛点、卖点、证明、异议处理和 CTA 分析。
3. 工具失败或未返回字幕时，可分析实际可见的标题、描述、封面或页面文字，并明确“未取得字幕”；不要把标题或推测口播冒充字幕。

## 8. 分页、配额与收集范围

- 搜索工具默认 `pageNum=1`、`pageSize=20`，最大 `pageSize=100`；实时 schema 若不同则以其为准。
- 先按完成结论所需的最小候选范围收集。用户要求“全部”时，只对用户明确指定的层级持续分页；按稳定 ID 去重，直到没有下一页、达到用户数量、到达服务上限、日期边界、配额/限流或服务端停止。
- 每一来源账本行记录请求条件、页码范围、每页大小、原始/去重条数、收集时间、币种/窗口、完成度和停止原因。第一页或额度中断的集合必须写“非完整结果”。
- 收到 `ERROR_API_MINUTE_MAX`、`ERROR_API_MONTH_MAX`、401、工具不可用或其它阻塞错误时停止该链路并报告已完成/未完成部分；不通过缩小、换凭证、重试轰炸或绕过限制来规避配额。

## 9. TikTok Creative Center：官方趋势补充

TikTok Creative Center 是当前官方趋势补充来源，不是 KSS MCP 的替代品。通过公开可访问的 TikTok-owned Creative Center 页面，在与 KSS 相同的国家/地区、行业/类目和可见时间筛选下，读取实际可见的热门产品、广告、关键词、视频、达人或趋势信号。

对每次读取记录：官方页面 URL、访问时间和时区、市场/行业/时间筛选、可见字段、样本范围与访问限制。将其作为当前趋势的补充信号，不能把页面摘要、缓存、搜索结果、记忆或第三方转载称为 Creative Center 数据，也不能把它与 KSS 的不同地区、币种或时间窗口直接合并。

页面需要登录、验证码、反爬、地区访问、不可见筛选或浏览器无法加载时，停止读取并在账本中写“读取失败”及原因；继续使用已有证据，不臆造官方趋势。

## 10. 跨来源验证与机会判断

### 来源账本与可比性

分析前建立来源账本：来源/工具或页面、获取时间（含时区）、市场、类目/关键词、请求和筛选、页数/样本、字段覆盖、币种与统计窗口、缺失/错误、完整性。将 KSS MCP、Creative Center 和浏览器来源分行记录，绝不混写。

先检查地区、类目映射、对象 ID、币种、统计窗口、样本选择和指标定义是否可比。不能对齐时分开展示并说明不能比较；缺失不是零，不能补估。浏览器可见内容仅是浏览器证据，不能伪装为 MCP 数据。

### 交叉验证规则

进入、需求、竞争、内容、达人和差异化结论必须至少由两个相互独立且口径兼容的来源/层级支持，例如：商品信号与关联店铺，商品信号与 ID 关联视频，视频模式与已确认达人，或 KSS 结果与同市场/类目的 Creative Center 可见趋势。若只有一个来源，只能写“待验证信号”，不能作 go/no-go 依据。

对每项关键结论列出支持证据、反证/缺口、来源账本编号和适用条件。冲突数据不能靠挑选有利样本解决；说明冲突、优先使用更直接且口径更完整的证据，或降级为待验证。

### 机会评分（不编造基准）

不用固定行业门槛、隐藏权重、虚构转化率或合成分数给机会排名。仅在已收集的同市场、同类目、同币种、同统计窗口样本内，使用明确、可审计的相对证据描述机会：

- 商品：价格带、需求/增长字段、评分/评论、供给与履约信号。
- 店铺：样本内可见的供给、服务、销售与内容/达人投入信号。
- 内容：ID 关联的商品/店铺、近期表现字段、可见主题和字幕/页面文字。
- 达人：地区/类目适配、已确认关联、可见内容模式与实际返回的表现字段。
- 官方趋势：同市场和行业筛选下可见的 Creative Center 当前信号。

将切入方向标为“优先验证”“条件进入”或“不建议当前进入”，并写清哪些跨来源证据和业务条件支持该标签。没有足够的可比数据时写“暂无法判断”，而不是计算排名。

## 11. 内容支柱与选题矩阵

从交叉验证的商品、视频、达人和字幕/可见文本中归纳内容支柱。每个支柱必须说明目标受众/场景、已观察到的痛点或购买理由、可复用的 Hook/证明方式、关联商品方向、适配达人类型、证据来源和不适用条件。

选题矩阵使用“内容支柱 × 用户场景/异议 × 证据方式 × 商品切入 × CTA 意图”组织。未取得字幕时仅根据可见文字描述，不声称已验证口播结构。不要为内容支柱规定无来源的比例、频次、时长、预算或预期表现；这些应作为待测试假设与调整条件。

## 12. 差异化定位

以已验证的供给、价格带、内容模式、达人覆盖和用户痛点寻找“可进入的空位”，而不是仅把竞品缺少的单一元素当作机会。定位需同时写出：目标人群/场景、核心承诺、可证明的商品或服务能力、内容表达、与代表样本的区别、支撑证据、待验证前提与不可复制/合规风险。

如果现有证据不能证明空位，建议验证路径（例如补充特定子类目、店铺、视频或用户反馈样本），不宣称差异化已成立。

## 13. 政策与商品可行性检查

在提出进入或推广建议前，检查目标市场当前适用的 TikTok/TikTok Shop 官方公开政策页面及用户提供的供应、履约、库存、资质信息。重点检查类目准入、受限/禁止商品、商品安全或标签/声明、知识产权、价格/促销陈述、达人商业披露、音乐/素材权利，以及跨境履约的已知限制。

只记录实际读取的官方页面和明确适用的要求；政策文本不清、市场不明、页面无法访问或商品资质未提供时，写“待确认”，不要凭记忆作合规结论。研究不替代法律、税务、平台或产品安全意见；不建议绕过平台控制、登录、验证码、反爬或地域限制。

## 14. 浏览器限制

浏览器只读取公开、可见的 TikTok、TikTok Shop、Creative Center 和官方政策页面信息。遇到登录、验证码、反爬、人机验证、地域/年龄限制、动态加载失败、访问控制或页面字段不可见时立即停止该页面的读取，记录 URL、时间、可见范围和失败原因。

不得使用浏览器绕过访问控制、抓取隐藏接口、伪造地区或把普通搜索/第三方页面冒充官方或 KSS 数据。浏览器不能证明平台未公开字段；无法取得的字段写“未公开”或“读取失败”。

## 15. 停止条件

在以下任一情况停止当前收集或深度判断，并记录已完成范围、停止原因、完整性与下一步：

1. 已获得支持范围内结论所需的跨来源证据，或完成用户明确要求的样本/层级。
2. 没有下一页、达到用户数量/日期窗口边界，或只剩重复/不相关结果。
3. 达到工具服务上限、配额/限流、401、工具不可用或服务端停止。
4. Creative Center、政策页面或 TikTok 页面遇到浏览器访问限制。
5. 目标类目或市场缺失，或关键缺失会改变结论且已提出一个聚焦问题。
6. 数据不足以形成交叉验证结论；交付标明限制的部分报告或“暂无法判断”，不以推测填补。

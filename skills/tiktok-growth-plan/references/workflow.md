# TikTok 增长计划工作流

本工作流把上游账号诊断和类目策略的**已取得证据**转为可执行计划。它不替代上游工作流，也不重新定义 KSS 工具、字段、浏览器读取或来源标准。

## 1. 必需输入门槛

必须依次取得以下四项：

1. **公开 TikTok 账号主页 URL**：交由 `tiktok-account-audit` 按其 URL 验证与规范化规则处理。
2. **目标类目**：用户希望进入、转型或增长的商品/内容类目。
3. **目标国家或地区**：所有类目研究、政策和市场口径的共同市场。
4. **商业目标**：例如验证进入、获得订单、建立内容资产或招募达人；不把模糊的“增长”替代为具体目标。

它们都是绝对门槛。一次只问一个：先缺账号 URL 时只问账号主页链接；随后只问类目、市场或商业目标中的第一个缺失项，并说明它会影响的判断。四项未齐时，禁止调用研究工具、启动上游研究、输出通用框架、条件策略、适配/进入判断、内容比例、30 天日历、KPI 或正式报告。

可选输入是预算、团队能力、库存、每日可投入时间、价格带、内容语言、计划周期和对标账号。只记录用户给出的值；未给出则为“未提供”，并在计划中把相关动作列为前提或待验证假设。

## 2. 账号诊断交接

完整复用 `tiktok-account-audit` 的 `SKILL.md` 和 `references/workflow.md`。不重述或改写其 `creator_profile`、`creator_videos`、`caption_extract` 或浏览器工作步骤。

从上游报告/来源账本带入：原始与规范化 URL、Handle、市场和样本窗口、请求及实际视频范围、来源与字段状态、账号类型和阶段、内容支柱/节奏、代表视频、可验证的 Hook/证明/CTA、商品或带货字段、可复用资产、主要约束、缺失和 A/B/C 可信度。保留每项结论的来源账本编号或证据链接；没有可验证字段时写“未提供”“未公开”或“读取失败”。

## 3. 类目策略交接

完整复用 `tiktok-category-strategy` 的 `SKILL.md` 和 `references/workflow.md`。不重述或改写其商品、店铺、视频、达人、字幕、Creative Center、政策或浏览器工具步骤。

从上游报告/来源账本带入：原始与规范化类目/子类目、目标市场、筛选和窗口、商品/店铺/视频/达人/官方趋势证据、进入结论及条件、价格和币种、内容模式、用户痛点、商品与达人机会、合规/供应风险、缺失与 A/B/C 可信度。单一来源或不可比数据仍只是待验证信号，不能升级为进入或适配事实。

## 4. 证据对齐与完整性

建立增长计划的交叉来源账本，将每条账号与类目证据保留在原来源下，并检查：

- 市场是否相同；不同时不比较表现或直接迁移策略。
- 类目映射、对象 ID、目标人群/场景和商业目标是否明确；歧义分开处理。
- 价格与币种、播放/互动/销量/销售额的定义和统计窗口是否兼容；不能对齐则分列并说明限制。
- 视频或达人是否有可验证的商品/店铺关联；没有关联时仅作为内容灵感，不作为交易能力证据。
- 账号侧与类目侧证据是否各自足以支持定位、进入决策和第一轮行动；缺少一侧时降低结论并触发一个关键问题。

不得把缺失当零、推测当事实、样本当全量，或以行业固定阈值计算伪精确“适配分”。每项判断应写明：观察、推断、支持证据、反证/缺口、适用条件和可信度。

## 5. 适配判断与差距矩阵

先形成一句话运营定位，再给“建议进入 / 条件进入 / 不建议当前进入 / 暂无法判断”。进入判断不只是类目机会：还必须同时考虑账号已有资产、目标市场、商业目标、供应/合规条件与执行资源。

用下表逐项匹配，不给没有证据支持的总分：

| 维度 | 账号当前能力或资产 | 类目要求/机会 | 适配观察与证据 | 差距、风险或反证 | 进入/行动影响 |
| --- | --- | --- | --- | --- | --- |
| 目标人群与场景 |  |  |  |  |  |
| 定位与价值承诺 |  |  |  |  |  |
| 商品/价格/库存与履约 |  |  |  |  |  |
| 内容支柱、Hook 与证明 |  |  |  |  |  |
| 达人合作与内容生产能力 |  |  |  |  |  |
| 市场、政策与素材合规 |  |  |  |  |  |

将每项标为“可保留资产”“待补齐能力”“阻断风险”或“待验证”。若存在未确认的类目准入、库存/履约、市场口径、商业目标或无法支撑主结论的证据缺口，不能给无条件进入建议。

## 6. 保留、停止和新增决策

将决定关联到差距矩阵和商业目标：

- **保留**：已有且可由目标类目证据支持的受众洞察、表达方式、内容公式、商品/达人关系或运营资产。
- **停止**：与目标定位冲突、不可衡量、占用关键资源、违反已知合规条件，或已有证据显示不能支撑目标的动作。数据不足时写“暂停验证”，不把低表现归因为单一原因。
- **新增**：为补齐明确差距而设的最小实验、能力、素材、商品验证或合规核验。写明依赖、负责人/资源、成功观察信号与停止条件。

不要因缺少预算或团队信息擅自指定采购、投放、达人数量、视频频次或承诺结果。

## 7. 三条增长路径

每条路径都从上游可验证证据出发，并标明假设、依赖、可观测指标、复盘节点和停止/转向条件：

1. **选品路径**：把类目商品/价格带/用户痛点与账号表达、库存、履约和合规能力匹配。先验证商品主张、证据材料与可售条件；商品、价格、销量、销售额字段分列，不能相互替代。
2. **视频路径**：把账号已有内容资产与类目中可验证的场景、Hook、证明、异议处理和 CTA 组合成有限实验。仅复用已取得的字幕或可见文字模式；未取得字幕时明确限制。
3. **达人路径**：以同市场、类目适配、可确认关联和可见内容模式筛选达人类型或合作假设。不得输出不必要的联系方式、承诺合作效果或建议绕过平台规则。

路径优先级由商业目标、差距、资源和证据强度决定。资源不足或关键依赖未确认时，将动作排为前置核验，不模拟不存在的执行能力。

## 8. 资源约束、30 天日历与内容配比

先将用户已知的预算、团队、库存和每日时间写入资源账本。再用四个阶段安排最小可行的实验与复盘：Day 1–7 证据/准备，Day 8–14 首轮测试，Day 15–21 复用或修正，Day 22–30 聚焦或停止。每个日历项必须有目标、路径、行动、资源/依赖、观察信号和调整条件。

“每周内容数量与比例”只能基于用户给出的时间/团队产能或明确写为**建议测试量**。分别列出内容支柱、对应商业意图、数量/比例、依据和调整条件；没有产能信息时不得伪装成已验证的最佳配比。

## 9. KPI、复盘和下一步提示词

按商业目标和可用字段选择少量 KPI，分别保留播放、互动、点击、商品加购、销量、销售额、成本等原始定义、来源和窗口；不能将其中任意指标互换为转化或 ROI。没有字段时写“暂不可测”，并设置补数或替代观察动作。

为每个路径设定固定复盘节点、比较条件与调整逻辑，例如仅在同市场、同窗口、相同内容/商品范围内比较；达到预先写明的学习条件时扩大，出现阻断风险、不可比数据、库存/合规问题或连续未验证假设时暂停/转向。不要使用未证实的行业阈值。

报告结尾生成可直接复制给 AI 的后续任务。每个提示词必须包含现有事实、数据来源/窗口、明确目标、限制、所需输出和“不得编造缺失数据”；优先覆盖补齐关键缺口、选品验证、视频脚本实验、达人筛选或复盘中的下一步。

## 10. 部分数据与停止条件

上游数据不完整时，先完成其允许的重试、分页、ID 关联、官方补充和公开浏览器降级。之后：

- 缺失会改变运营定位、进入结论或首要行动时，只问一个最关键的问题并停止深度判断。
- 用户明确要求继续时，交付部分报告：所有受影响的结论标为待验证，列出缺失状态、影响、已尝试补救、下一步和 A/B/C 可信度。
- 仅在已有证据足以支持其适用范围内的结论时停止收集；或在没有下一页、达到用户样本/日期上限、只剩重复/无关结果、工具配额/限流/服务端停止、浏览器受限、关键输入缺失或关键缺口未解决时停止。

绝不绕过登录、验证码、反爬、地域限制或访问控制，也不执行发布、私信、下单、店铺修改或广告投放等外部写操作。

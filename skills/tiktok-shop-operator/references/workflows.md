## 通用执行规则

1. 先从用户请求提取地区、关键词/类目、价格、评分、销量、销售额、增长率、统计周期、数量和排序。
2. 优先使用 ID 关联对象；名称仅用于查询入口或展示。
3. 保留用户给出的全部筛选条件，不静默放宽条件。
4. 缺少会阻止有效查询的条件时，只追问一个最关键问题。
5. 用户要求“全部”时，以最大允许 `pageSize=100` 分页，直到无下一页、达到服务上限或额度中断。
6. 用户指定排序时遵守；否则在已获取范围内按相关销售额降序。
7. 工具没有列出合法 `orderField` 时，不发明值；先查询，再对返回数据本地排序。
8. 每次输出都说明查询口径、分页范围、结果数量和完整性。
9. 只使用 MCP 返回值或能够由明确 ID/Handle 构造的链接；缺失内容写“未提供”。
10. 结束时给出基于证据的结论、风险与下一步动作。
11. 增长率输入单位未在公开文档中定义时，先检查实时工具 schema 或官方示例，不得猜测 50% 对应 `50` 还是 `0.5`。

## 商品选品

### Required inputs

- 地区；未指定时可使用 MCP 默认 US，但必须披露。
- 类目或关键词；泛类目任务可通过关键词起步。
- 用户给出的价格、评分、销量、增长率、上架时间等条件。
- 统计周期与返回数量。

### Tool sequence

1. 调用 `product_search`。
2. 将自然语言范围转为 `price`、`rating`、`salesVolumeLst7d`、`salesVolumeGr7d`、`salesVolumeLst30d` 或 `salesLst30d`。
3. 需要排除词时使用 `excludeKeyword`。
4. 需要补充店铺信息时记录商品的 `shopId`，进入“店铺分析”。
5. 需要验证内容生态时记录商品 `id`，进入“爆款视频”。

### Pagination and sorting

- 普通任务先取用户要求的最小结果范围。
- “全部”任务持续翻页并去重 `product.id`。
- 优先按用户要求排序；未指定时按对应周期销售额降序。
- 无合法服务端排序枚举时，在完整的已获取范围内本地排序。

### Validation

- 检查地区、币种和统计周期。
- 区分销量增长率与销售额，不能互相替代。
- 评分条件只使用 `rating`，不能用评论数近似。
- 若商品链接没有返回，写“未提供”。

### Output

使用商品结果模板，附上查询口径、完整性和选品建议。

### Stop conditions

- 无下一页。
- 达到用户数量要求。
- 达到工具上限。
- 返回 `ERROR_API_MINUTE_MAX`、`ERROR_API_MONTH_MAX`、401 或其它阻塞错误。

## 店铺分析

### Required inputs

- 店铺 ID、名称、商品返回的 `shopId`，或地区与类目条件。
- 评分、平均价格、近 30 天销量/增长等可选条件。

### Tool sequence

1. 已知店铺 ID 时调用 `shop_search`，使用 `searchType=N` 和店铺 ID。
2. 只有名称时使用 `searchType=K` 与关键词。
3. 需要查看店内商品时调用 `product_search.shopId`。
4. 需要内容表现时调用 `video_search.shopId`。

### Pagination and sorting

- 多店铺任务按用户要求分页。
- 未指定排序时，按 `salesLst30d` 降序在已获取范围排序。

### Validation

- 通过 `shopId` 确认同名店铺。
- 分开展示店铺总销量与近 30 天销量。
- 店铺链接优先使用 MCP 返回的 `link`。

### Output

使用店铺结果模板，总结规模、增长、服务指标、内容供给和风险。

### Stop conditions

遵循通用停止条件；店铺 ID 精确查询获得唯一匹配后停止。

## 爆款视频

### Required inputs

- 商品 ID、店铺 ID、关键词或类目。
- 发布时间、播放量、互动率、销量、销售额、价格或时长条件。
- 用户对“爆款”的明确定义；未定义时根据可用销售额、销量与互动指标综合展示，不擅自设置隐藏阈值。

### Tool sequence

1. 调用 `video_search`。
2. 商品关联使用 `productId`；店铺关联使用 `shopId`。
3. 只要带货视频时设置 `product=true`。
4. 记录 `creator.id`、`creator.handleName` 和视频 `id`。
5. 需要达人深挖时进入“达人匹配”。
6. 需要脚本研究时构造可验证的视频 URL，再进入“字幕分析”。

### Pagination and sorting

- 用户指定销售额时按 `salesLst30d` 降序。
- 用户指定销量时按对应销量字段降序。
- 未指定时优先按近 30 天销售额降序，并说明使用的统计周期。

### Validation

- 不把播放量高直接等同于带货销售额高。
- 视频、商品和达人指标分别保留各自统计口径。
- 只有 Handle 与视频 ID 均存在时构造 TikTok 视频链接。

### Output

使用视频结果模板，并总结钩子、商品、达人规模与销售表现之间的关系。

### Stop conditions

遵循通用停止条件；达到用户需要的视频数量后停止。

## 达人匹配

### Required inputs

- 地区、关键词、达人分类或带货类目。
- 粉丝量、平均播放、互动率、增长率、销量、GPM 等条件。
- 从视频获得的达人 Handle/昵称与预期达人 ID。

### Tool sequence

1. 泛筛选调用 `creator_search`，使用类目与数值范围。
2. 从视频关联达人时，使用 `creator.handleName` 或 `creator.name`，设置 `searchType=N`。
3. 查询返回后，将 `creator_search.id` 与视频中的 `creator.id` 对照。
4. ID 无法确认时标记“关联待确认”，不得把同名达人当作已确认对象。

### Pagination and sorting

- 用户指定指标时按该指标排序。
- 未指定时按带货销量降序；如需销售额但工具未返回销售额，明确说明并使用现有指标，不做估算。

### Validation

- 区分粉丝量、平均播放、互动率和带货表现。
- 联系方式只展示 MCP 返回内容。
- 达人链接只在 `uniqueId` 或 Handle 可用时构造。

### Output

使用达人结果模板，给出优先级、匹配理由、内容适配性和联系风险。

### Stop conditions

遵循通用停止条件；精确达人 ID 经交叉确认后可停止。

## 字幕分析

### Required inputs

- 单个可访问的 TikTok 视频 URL。
- 用户需要原文、翻译、摘要、钩子、卖点或 CTA 中的哪些内容。

### Tool sequence

1. 调用 `caption_extract`，传入视频 URL。
2. 保留工具返回的原始字幕。
3. 在原始字幕基础上做分段、摘要和结构分析。
4. 用户要求翻译时，将翻译与原文分开标记。

### Pagination and sorting

字幕提取不分页。多个视频时逐个处理并记录成功、失败与未执行数量。

### Validation

- 不能把视频标题或描述冒充字幕。
- 工具未返回字幕时说明失败原因，不补写“可能的字幕”。
- 引用原文时保持顺序和语义。

### Output

使用字幕结果模板，输出原文、分段摘要、开头钩子、卖点、证明、反对意见处理和 CTA。

### Stop conditions

单个 URL 完成或失败后停止；批量任务遇到额度错误时停止剩余调用并报告进度。

## 完整运营闭环

### Required inputs

- 目标地区与类目/关键词。
- 核心筛选条件与业务目标。
- 需要的候选数量、统计周期和最终交付形式。

### Tool sequence

1. 用 `product_search` 建立商品候选池。
2. 用 `shop_search` 补充候选店铺信息。
3. 用 `video_search.productId` 或 `video_search.shopId` 查找关联带货视频。
4. 用视频返回的 Handle/昵称调用 `creator_search`，并通过达人 ID 交叉确认。
5. 对入选视频 URL 调用 `caption_extract`。
6. 汇总商品、店铺、视频、达人和字幕证据。
7. 输出优先级、内容参考、达人合作建议、风险和下一步行动。

### Pagination and sorting

- 每一层先满足用户的候选数量，再进入下一层，控制额度。
- 用户要求“全部”时只对明确指定的层级全量分页；其它层使用经过排序的候选集。
- 记录每层查询页数、返回条数和筛选后数量。

### Validation

- 每条结论至少指向一个明确的 MCP 证据字段。
- 不把不同币种、地区和周期直接合并排名。
- 关联对象必须保留 ID；不确定关联要显式标记。
- 无字幕时仍可完成数据分析，但不能声称已分析视频口播。

### Output

依次输出查询口径、核心结果表、证据解读、风险、优先级和 3–5 条下一步动作。

### Stop conditions

- 完成用户要求的闭环。
- 必要工具不可用。
- 额度或限流中断。
- 关键关联无法确认且继续调用不会提高可信度。

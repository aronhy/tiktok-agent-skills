## Example 1：单工具选品

**用户请求**

> 查找美国区评分 4.5 以上、价格 30–50 美元、近 7 天增长率超过 50% 的美妆商品，按 7 天销售额降序。

**工具计划**

1. 调用 `product_search`。
2. 使用 `region=US`、`rating.from=4.5`、`price.from=30` 和 `price.to=50`。
3. 检查实时工具 schema 或官方示例，确认“增长率 50%”对应的 `salesVolumeGr7d.from` 输入单位；不得猜测为 `50` 或 `0.5`。
4. 文档未给出合法销售额排序枚举时，在已获取范围按 `salesLst7d` 本地降序。
5. 输出查询口径、实际增长率输入值及商品模板。

## Example 2：商品到视频到达人

**用户请求**

> 分析 PolyLemon 自动磁力搅拌杯的带货视频和达人，按销售额排序。

**工具计划**

1. 用 `product_search` 确认商品 ID 与店铺 ID。
2. 用 `video_search.productId` 查询视频。
3. 按 `salesLst30d` 对已获取视频降序。
4. 使用视频返回的达人 Handle/昵称调用 `creator_search`。
5. 用达人 ID 交叉确认关联。
6. 输出商品、视频、达人和行动建议。

## Example 3：字幕提取

**用户请求**

> 提取这个 TikTok 视频的字幕，并分析开头钩子、卖点和 CTA：https://www.tiktok.com/@example/video/123

**工具计划**

1. 调用 `caption_extract`。
2. 保留原始字幕。
3. 使用字幕模板做结构分析。
4. 失败时报告工具错误，不根据标题补写字幕。

## Example 4：全部结果分页

**用户请求**

> 查找符合条件的全部商品。

**工具计划**

1. 先确认“符合条件”指向的筛选条件。
2. 设置 `pageSize=100`。
3. 从 `pageNum=1` 开始分页并按商品 ID 去重。
4. 无下一页时停止。
5. 如果限流或额度中断，使用非完整结果模板，不能声称“全部”。

## Example 5：销售额降序

**用户请求**

> 把这些视频按近 30 天销售额降序输出。

**工具计划**

1. 使用 `video_search` 获取 `salesLst30d`。
2. 工具支持明确排序值时服务端排序；否则对完整的已获取范围本地排序。
3. 在查询口径中说明排序字段、范围和统计周期。

## Example 6：空结果

**用户请求**

> 查找美国区评分 5.0、价格 1–2 美元、近 7 天销量超过 10 万的商品。

**工具计划**

1. 原样执行筛选。
2. 无结果时使用空结果模板。
3. 建议可放宽的单个条件，但不自动重查。

## Example 7：MCP 未连接或 401

**用户请求**

> 帮我查美国美妆店铺。

**工具计划**

1. 如果工具不可见，说明 MCP 未连接或客户端需刷新。
2. 如果返回 401，检查 `secret-key` 与 MCP 权限。
3. 不使用网络搜索结果冒充 MCP 数据。

## Example 8：额度中断

**用户请求**

> 查询全部美国美妆商品。

**工具计划**

1. 分页查询并记录每页条数。
2. 收到 `ERROR_API_MONTH_MAX` 后立即停止。
3. 报告已完成页数、收集条数和未完成范围。
4. 使用非完整结果模板。

## Example 9：缺少返回字段

**用户请求**

> 给出每个达人精确销售额。

**工具计划**

1. 检查 `creator_search` 返回字段。
2. 当前工具没有达人销售额字段时明确写“未提供”。
3. 可以展示带货销量和 GPM，但不能反推精确销售额。

## Example 10：缺少必要条件

**用户请求**

> 帮我找适合合作的达人。

**工具计划**

1. 判断地区是当前最影响结果的缺失项。
2. 一次只追问一个关键问题：“目标地区是什么？”
3. 得到回答后，再判断是否还必须追问产品类目；每次仍只问一个问题。
4. 条件足够后调用 `creator_search`。
